# Arrays and Objects

## What is an Array?

An array is a data structure that stores multiple values in a single variable. It helps organize related data together so it can be accessed and managed easily.

For example, instead of creating separate variables for each goal, I can store all my goals inside one array.

---

## Why Do Arrays Exist?

Most applications need to manage many similar items rather than just one.

Examples:

- Instagram stores many posts.
- Amazon stores many products.
- WhatsApp stores many messages.
- Decision Journal stores many goals and reflections.

Arrays make it possible to work with these collections efficiently.

---

## Array vs Object

### Array

An array stores a collection of similar items.

Example:

```javascript
const colors = ["Blue", "Green", "Black"];
```

Arrays use index numbers to access elements.

---

### Object

An object stores related information about a single entity using key-value pairs.

Example:

```javascript
const goal = {
    title: "Learn MERN",
    progress: 20,
    completed: false
};
```

Objects use property names to access values.

---

## Why Use an Array of Objects?

An array of objects combines the strengths of both data structures.

The array stores multiple items, while each object represents one complete item with its own properties.

Example:

```javascript
const goals = [
    {
        title: "Learn MERN",
        progress: 20,
        completed: false
    },
    {
        title: "Solve DSA",
        progress: 45,
        completed: false
    }
];
```

This structure is commonly used in modern web applications and databases.

---

## Where Will I Use Arrays in My Decision Journal?

Arrays will store collections of data such as:

- Goals
- Reflections
- Milestones
- Notifications

Each item in these arrays will be represented as an object.

This structure will later be used in React state and MongoDB collections.

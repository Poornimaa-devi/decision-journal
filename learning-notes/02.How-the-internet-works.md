# How the Internet Works

## What is the Internet?

The Internet is a worldwide network of computers that communicate with one another. It allows people to access websites, send messages, share information, and use online services from anywhere in the world.

---

## What is a Browser?

A browser is software that allows users to access websites. It sends requests to servers, receives responses, and displays web pages in a user-friendly format.

Examples include Google Chrome, Microsoft Edge, Firefox, and Safari.

---

## What is a Server?

A server is a computer or program that stores application data and responds to requests from users. It processes information and sends the required data back to the browser.

---

## What is a Client?

The client is the user's device or browser that requests information from the server. It is responsible for displaying the application's interface.

---

## What is a Request?

A request is a message sent from the browser to the server asking for information or asking the server to perform an action.

Example:
When I click "Create Goal", my browser sends the goal details to the server.

---

## What is a Response?

A response is the server's reply to the client's request. It tells the browser whether the operation was successful and may include additional data.

Example:
The server confirms that the goal has been saved successfully.

---

## How will these appear in my Decision Journal project?

When a user creates a goal:

1. The user fills out the form.
2. The browser sends an HTTP request.
3. The backend receives the request.
4. The backend validates the data.
5. The backend stores the goal in MongoDB.
6. MongoDB confirms the data is saved.
7. The backend sends a success response.
8. The browser updates the dashboard and displays the new goal.

This interaction happens every time users create, update, delete, or view information in the application.

# Update Operation

## What is the Update Operation?

The Update operation modifies existing data without creating a new record or deleting the old one.

In my Decision Journal project, updating means changing information inside an existing goal object.

---

## Why Update Instead of Delete and Recreate?

Updating is more efficient because only the required information changes.

If I delete and recreate a goal:

- The original data may be lost.
- References to that goal may break.
- The operation becomes less efficient.

Updating preserves the existing object while changing only the required fields.

## Why Update Data Before UI?

The goals array is the source of truth.

The correct workflow is:

Update Data

↓

Render UI

If I update the UI first without updating the data, the application will become inconsistent.

---

## How Does renderGoals() Help After an Update?

After modifying a goal object, the UI still shows the old information.

Calling `renderGoals()` clears the old UI and rebuilds it using the updated data from the goals array.

This keeps the webpage synchronized with the application's data.

# Week 2 Review

## How Does Data Flow Through My Application?

The application follows a simple flow. The user enters goal details in the form and clicks the Create Goal button. JavaScript reads the input values, creates a goal object, and stores it in the goals array. After updating the array, the `renderGoals()` function is called to regenerate the goal list and update the webpage.

---

## What Happens When the User Creates a Goal?

1. The user fills out the form.
2. The submit event is triggered.
3. `preventDefault()` stops the page from refreshing.
4. JavaScript reads the input values.
5. A new goal object is created.
6. The object is added to the goals array.
7. `renderGoals()` is called.
8. The webpage updates and displays the new goal.
9. The form is cleared for the next input.

---

## Why Is Rendering Important?

Rendering converts JavaScript data into visible HTML elements. Instead of manually editing the webpage every time data changes, the application simply updates the data and re-renders the user interface.

---

## Why Do We Separate Data from UI?

Keeping data separate from the UI makes the application easier to maintain and update. The goals array stores the application's data, while the render function is responsible only for displaying it. This separation reduces bugs and improves code organization.

---

## Which JavaScript Concept Was the Hardest This Week?

The most challenging concept was understanding rendering and why the existing UI must be cleared before generating it again. After practicing with `renderGoals()`, I understood that this prevents duplicate content and keeps the webpage synchronized with the data.

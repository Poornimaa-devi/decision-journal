# Day 19 – Sorting

## What is Sorting?

Sorting is the process of arranging data in a specific order. It changes only the order of the data, not the data itself.

Example:

Before:
- Learn React
- AWS Certification
- Solve DSA

After (A–Z):
- AWS Certification
- Learn React
- Solve DSA

---

## Ascending vs Descending

Ascending arranges values from smallest to largest or A to Z.

Examples:
- A → Z
- 1 → 100
- Oldest → Newest

Descending arranges values from largest to smallest or Z to A.

Examples:
- Z → A
- 100 → 1
- Newest → Oldest

---

## Search vs Filter vs Sort

### Search
Search displays only items that match the entered text.

Example:
Searching "AWS" shows only AWS-related goals.

### Filter
Filter displays items that match a selected condition.

Example:
Show only High Priority goals.

### Sort
Sort changes the order of the displayed goals without removing any.

Example:
Display goals alphabetically.

---

## Why shouldn't we sort the original array?

The original goals array should always remain unchanged because it is the main source of data.

Instead:
- Create a copy of the array.
- Sort the copy.
- Display the sorted copy.

This prevents unexpected bugs and makes the application easier to maintain.

---

## How does sorting fit into my application?

Workflow:

Goals Array
      ↓
Search
      ↓
Filter
      ↓
Sort
      ↓
Display Goals

Sorting is the final step before displaying the goals to the user.

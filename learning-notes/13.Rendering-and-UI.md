# Rendering and UI

## What is Rendering?

Rendering is the process of converting JavaScript data into visible HTML elements that users can see on the webpage.

For example, my goals are stored in a JavaScript array. The render function reads this array and creates HTML elements to display each goal.

---

## Why Do We Render from Data?

The JavaScript array is the source of truth for my application.

Instead of manually editing HTML every time data changes, I update the array and let the render function rebuild the user interface.

This keeps the data and UI synchronized.

---

## Why Create a Render Function?

A render function keeps all UI update logic in one place.

Whenever the goals array changes, I simply call the render function instead of writing duplicate code.

This makes the application easier to maintain and understand.

---

## What is Re-rendering?

Re-rendering means rebuilding the visible UI after the application data changes.

Whenever a new goal is added, the render function clears the old content and generates an updated list from the goals array.

---

## Why Clear Old UI Before Rendering Again?

If I don't clear the existing content, every render will add duplicate goal cards to the page.

Clearing the container first ensures that only the latest data is displayed.

---

## How Will React Improve This Process?

In this project, JavaScript redraws every goal whenever data changes.

React improves this by using the Virtual DOM, which updates only the parts of the webpage that actually changed.

This makes applications faster and more efficient.

---

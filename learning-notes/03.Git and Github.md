# Git and GitHub

## Why Was Git Invented?

As software projects grew larger, developers faced challenges in managing changes, collaborating with others, and recovering from mistakes. Git was created to solve these problems by tracking every change made to a project and allowing developers to return to earlier versions whenever necessary.

Instead of creating folders like Project_Final_v2_Final, Git maintains a complete history of the project in an organized way.

---

## Git vs GitHub

Git is a version control tool installed on my computer. It tracks changes, records commit history, and manages different versions of my project.

GitHub is an online platform that stores Git repositories. It acts as a backup of my project and makes collaboration with other developers easier.

In simple terms:

- Git manages versions locally.
- GitHub stores those versions online.

---

## What is a Repository?

A repository is the complete workspace of a project.

It contains:

- Source code
- Documentation
- Images
- Configuration files
- Commit history

For this project, the `decision-journal` folder is my repository.

---

## What is a Commit?

A commit is a snapshot of the project at a particular point in time.

Each commit records the changes made since the previous commit along with a descriptive message explaining those changes.

Instead of thinking of a commit as "Save," I think of it as creating a milestone in the project's history.

---

## What is a Branch?

A branch is an independent line of development.

It allows developers to work on new features or fix bugs without affecting the main version of the project.

Once the work is complete and tested, the branch can be merged back into the main branch.

---

## How Git Will Help My Decision Journal Project

Git will help me safely develop my Decision Journal application by keeping track of every change I make.

If I introduce a bug or accidentally delete important code, I can recover an earlier version of the project.

Git will also allow me to experiment with new features using branches without risking the stability of the main application.

---

## Thinking Exercise

### Without Git

If I spent five days developing the login page and accidentally deleted everything, I would either have to rebuild the feature from scratch or hope that I had created a backup manually.

---

### With Git

I can simply view the commit history, identify the last working version, and restore the project to that state. This saves time, reduces stress, and prevents loss of work.

---

## Interview Question

### Why do software companies use Git instead of saving folders like Project_Final_v2_Final?

Saving multiple folders creates confusion because it becomes difficult to identify which version is the latest or what changes were made.

Git keeps a structured history of every change, records who made the changes, allows developers to collaborate safely, and makes it easy to restore previous versions whenever necessary.

This organized workflow makes software development faster, more reliable, and easier to manage.

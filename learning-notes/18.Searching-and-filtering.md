# Day 18 - Search and Filter

## Difference Between Search and Filter

Search is used to find specific data by matching text entered by the user.

Filter is used to display data that meets a selected condition, such as priority or completion status.

Search looks for keywords, while filtering selects data based on predefined categories.

---

## Why Shouldn't Search Modify the Original Array?

The original goals array is the source of truth for my application.

If I modify it during searching, I may lose data and won't be able to return to the complete list.

Instead, I create a temporary list of matching goals and display that.

---

## What is Live Search?

Live Search updates the displayed results immediately as the user types.

The user does not need to press a Search button or reload the page.

JavaScript listens for the input event and updates the displayed goals instantly.

---

## Why are Search and Filter Different?

Search uses text entered by the user to find matching goals.

Filter displays goals that satisfy a selected condition such as High, Medium, or Low priority.

Both can be used together to narrow down results.

---

## How Does My Application Display Filtered Results?

1. Read the search text.
2. Read the selected priority.
3. Find matching goals from the original goals array.
4. Display only the matching goals using renderGoals().

---

## Key Takeaways

- Search finds matching text.
- Filter selects matching categories.
- The original data should never be modified.
- Live search improves user experience.
- Rendering displays the filtered results.

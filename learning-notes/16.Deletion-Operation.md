# Day 16 - Delete Operation

## What is the Delete Operation?

The Delete operation is one of the four CRUD operations. It removes existing data from an application. In the Decision Journal project, deleting a goal removes it from the goals array and updates the webpage.

---

## Why Delete Data Before Updating the UI?

The goals array is the source of truth for the application.

When a goal is deleted, it should first be removed from the array. After that, the user interface is re-rendered based on the updated array.

This keeps the data and UI synchronized.

---

## Why Show Confirmation Dialogs?

Deleting data is a permanent action.

A confirmation dialog helps prevent users from accidentally deleting important information. It gives users one final chance to cancel the action.

---

## What is the Role of renderGoals() After Deletion?

The renderGoals() function redraws the list of goals after one has been deleted.

Instead of manually removing HTML elements, the application regenerates the entire goal list from the updated array.

This keeps the UI accurate and consistent.

---

## Why Are IDs Better Than Indexes?

Indexes represent the position of an item in an array.

If items are added or deleted, indexes change.

IDs remain unique and never change, making them a reliable way to identify data.

Databases such as MongoDB use unique IDs for this reason.

---

## Key Takeaways

- Delete removes unwanted data.
- Update the data before updating the UI.
- Confirmation dialogs improve user experience.
- renderGoals() keeps the UI synchronized with the data.
- IDs are more reliable than indexes.

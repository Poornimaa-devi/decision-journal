# Day 17 - Local Storage

## What is Local Storage?

Local Storage is a feature provided by the browser that allows small amounts of data to be stored on a user's computer. The stored data remains available even after refreshing the page, closing the browser, or restarting the computer.

---

## Difference Between Memory and Storage

### Memory
- Temporary storage used while the program is running.
- Data is lost when the page is refreshed or closed.

### Storage
- Permanent storage inside the browser.
- Data remains available until the user clears it.

In my Decision Journal project, the goals array was previously stored only in memory. Today, I started storing it in Local Storage.

---

## Why Data Disappeared Before Today

Earlier, my goals were stored only inside the JavaScript array.

Whenever the browser refreshed, JavaScript restarted and created a new empty array. As a result, all previously entered goals disappeared.

---

## What is JSON?

JSON stands for JavaScript Object Notation.

It is a text format used to store and transfer structured data such as objects and arrays.

JSON makes it possible to save JavaScript objects as strings.

---

## Why Convert Objects to JSON?

Local Storage can only store strings.

Since my goals are stored as an array of objects, I must convert them into a JSON string before saving them.

When loading the data, I convert the JSON string back into a JavaScript array.

---

## How Does My Application Load Saved Goals?

1. The webpage opens.
2. JavaScript checks Local Storage.
3. If saved goals exist, they are loaded.
4. JSON is converted back into an array.
5. The goals array is restored.
6. renderGoals() displays the saved goals.

---

## Key Takeaways

- Local Storage keeps data after refreshing the page.
- Memory is temporary while storage is permanent.
- JSON is used to convert objects into strings.
- My application now remembers user goals.

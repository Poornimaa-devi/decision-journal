# Variables and Data Types

## What is a Variable?

A variable is a named container that stores information in a program. It allows us to save data and use it whenever needed without writing the same value repeatedly.

For example, in my Decision Journal project, I can store the user's name, goal title, or progress in variables.

---

## Why Do Variables Exist?

Variables make programs flexible and reusable. Instead of hardcoding values, we can store them in variables and update them whenever required.

Without variables, changing data would require editing the code repeatedly.

---

## Primitive Data Types

Primitive data types store a single value.

Examples:

- String → "Learn MERN"
- Number → 20
- Boolean → true
- Null → No value assigned
- Undefined → Value not yet assigned

---

## Non-Primitive Data Types

Non-primitive data types can store collections of values.

Examples:

- Object
- Array

Objects group related information together, while arrays store lists of items.

---

## Scope

Scope determines where a variable can be accessed in a program.

Variables declared inside a function can only be used within that function, while variables declared outside are available in a larger part of the program.

Understanding scope helps avoid accidental errors and keeps code organized.

---

## Why Are Objects Useful?

Objects allow us to represent real-world entities by grouping related information together.

Instead of creating separate variables for a goal's title, deadline, progress, and completion status, we can store them inside a single Goal object.

This makes the code cleaner and easier to manage.

---

## Where Will I Use Objects in My Decision Journal?

Objects will represent important entities such as:

- User
- Goal
- Reflection
- Milestone

Later, these objects will become React state and MongoDB documents, making it easier to manage application data.

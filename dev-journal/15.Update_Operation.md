# Day 15 – Update Operation

## Today's Goal

Learn how to update existing goals and keep the user interface synchronized with the updated data.

---

## What I Built

I added an Edit button for every goal.

When the user clicks Edit, a prompt appears asking for a new goal title.

After entering a new title, the goal object is updated and the UI is re-rendered.

---

## Biggest Challenge

Understanding how JavaScript knows which goal should be updated was initially confusing.

Using the goal index helped me identify the correct goal.

---

## How the Update Flow Works

1. User clicks Edit.
2. JavaScript identifies the selected goal.
3. A prompt asks for a new title.
4. The goal object is updated.
5. `renderGoals()` is called.
6. The updated goal appears on the page.

---

## What I Learned About Modifying Data

I learned that data should always be updated first.

The UI should simply display whatever exists in the data model.

This keeps the application organized and easier to maintain.

---

## Reflection

Today's lesson helped me understand how real applications modify existing information without recreating everything from scratch.

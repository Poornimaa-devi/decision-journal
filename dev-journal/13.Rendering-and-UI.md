# Day 13 – Rendering and UI

## Today's Goal

Understand how rendering works and learn why modern applications generate the user interface from data instead of manually updating HTML.

---

## What I Built

I improved my goal rendering logic by creating a dedicated `renderGoals()` function.

Whenever a new goal is created, the goals array is updated and the entire goal list is rendered again.

---

## Biggest Learning

I learned that the data should always be the source of truth.

Instead of directly changing the webpage, I should update the data first and then re-render the UI.

---

## Most Difficult Concept

Understanding why the page clears all existing goals before displaying them again was initially confusing. After practicing, I realized it prevents duplicate content.

---

## How Rendering Changed My Thinking

Earlier, I thought webpages should be updated manually.

Now I understand that professional applications update the data first and let the UI regenerate automatically.

This approach makes applications easier to maintain and prepares me for learning React.

---

## Reflection

Today's lesson helped me understand one of the most important ideas in frontend development: keeping data and the user interface separate.

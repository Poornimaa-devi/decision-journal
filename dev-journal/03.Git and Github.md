# Day 3 – Git and GitHub

## Today's Goal

Understand the purpose of Git, how it differs from GitHub, and how version control supports professional software development.

---

## What I Learned

Today I learned that Git is much more than a backup tool. It is a version control system that records the complete history of a project.

I also understood the difference between Git and GitHub. Git works locally on my computer to track changes, while GitHub stores my repository online and enables collaboration.

I learned that commits should represent meaningful milestones rather than random saves, and branches provide a safe environment for developing new features without affecting the main project.

---

## One Concept That Surprised Me

The idea that a commit is not simply saving a file but creating a permanent checkpoint in the project's history surprised me. I also found it interesting that developers can easily return to any previous version of their project using Git.

---

## How Git Fits Into Professional Development

Git is an essential tool in professional software development because it helps teams collaborate efficiently, manage project history, recover from mistakes, and organize feature development through branches.

Every software company uses version control to ensure that projects remain stable, maintainable, and easy to develop over time.
